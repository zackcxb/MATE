# Agent 自复盘与资产沉淀

## 目标

把本次任务中的隐性决策、取舍逻辑、排查路径显性化，并把可复用经验沉淀为项目资产。

## 固定触发词

`启动自复盘`、`执行资产沉淀`、`复盘本次任务`。

## 适用场景

1. 里程碑节点达成后/重大架构变更落地/需求闭环后。
2. 显式声明 incident scope 的复杂问题收口后。
3. 用户主动要求定向复盘且明确复盘范围时。

## 前置铁则

1. 复盘必须通过独立 `agent-self-audit` subagent 执行，主上下文只接收摘要结果。
2. 所有结论必须来自本次任务的真实证据（commit、测试、审查记录）。
3. 对 `AGENTS.md`、`docs/project-context.md`、`skills/*.md` 仅做增量更新。
4. 复盘必须显性化“决策链 + 调查链 + 失败与恢复”。

## 标准执行步骤

#### 步骤1：Sub-Agent内完成全链路行为回溯（仅独立上下文运行，不传入主对话）
在执行任何文件写入前，必须先完成全流程回溯，输出结构化回溯底稿存储在/docs/retros。，作为后续所有沉淀工作的唯一依据，回溯必须完整覆盖以下维度：
1.  需求锚定回溯：原始需求核心目标、验收标准、约束条件；我对需求的初始理解、是否出现偏差、如何修正对齐、最终锁定的需求边界
2.  全决策链拆解：完整还原从需求到成果的每一个关键决策点，包括技术选型、代码设计、工具调用、风险规避的决策逻辑、备选方案放弃原因、最终验证效果
3.  执行过程复盘：计划路径vs实际执行路径的偏差、偏差原因与修正方案；本次任务遇到的所有报错、卡点、阻塞问题，完整排查思路、试错过程、最终解决方案、可复用避坑经验
4.  成果验证复盘：最终交付是否100%满足原始需求、验证方式与覆盖场景、未覆盖的边界情况、遗留待办、后续优化方向

#### 步骤2：Sub-Agent内完成4项标准化资产沉淀（仅写入对应文件，不污染主上下文）
##### 模块1：增量更新/生成项目AGENT.MD
- 文件定位：本项目AI Agent执行编码、协作、项目管理任务的唯一执行准则，沉淀验证有效的最佳实践，淘汰无效/错误行为模式
- 核心更新内容：仅新增/修正本次任务验证有效的内容，包括需求处理规范、编码执行规范、工具使用规范、项目管理规范、风险控制规范、复盘沉淀规范
- 存储路径：项目根目录 `./AGENT.MD`

##### 模块2：增量更新项目context.md
- 文件定位：项目唯一全局上下文入口，确保Agent冷启动快速对齐项目全貌，避免上下文丢失、信息断层
- 核心更新内容：本次任务核心进展、已完成事项、待办事项更新、依赖与风险变更、上下文变更记录
- 存储路径：`docs/project-context.md`，无则自动按标准结构创建

##### 模块3：提取可复用行为模式，生成独立标准化Skill
- 核心要求：**仅在必要时**从本次任务中提取可跨项目/跨任务复用的、解决特定场景问题的标准化执行流程
- 存储路径：项目 `./skills/` 目录，命名格式：`skill-xxx-xxx.md`（全小写，横杠分隔）
- 去重规则：若已有同名Skill，仅增量更新优化执行步骤、注意事项、实践案例，不重复创建

##### 模块4：自迭代优化本复盘Skill
- 核心要求：基于本次复盘执行的实际效果，增量更新本Skill的执行步骤、避坑规则、优化点，确保本Skill的能力持续迭代
- 禁止操作：不得修改本Skill的核心定位，仅可优化执行细节、补充边界规则

## 频率守卫

1. 默认每个里程碑最多执行一次复盘。
2. 仅当存在显式 incident scope 时，允许里程碑内追加一次专项复盘。
3. 固定触发词不等于自动触发；需满足场景条件与频率守卫。

## 核心交付物

1. 复盘执行摘要（主对话可读）。
2. 文件变更清单（路径 + 核心变更点）。
3. 可复用实践提炼（最多 3 条）。
4. 下一次任务的优化建议。

## 红线

1. 禁止在主对话输出完整复盘底稿或文件全文。
2. 禁止全量覆盖历史有效内容。
3. 禁止沉淀无证据支撑的空泛规则。
4. 禁止绕过 subagent 直接在主上下文执行全量复盘。
