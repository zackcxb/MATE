# 复盘到 AGENTS 规则沉淀

## 目的

把复盘中的高价值经验沉淀为稳定规则，同时避免把一次性细节写入 `AGENTS.md`。

## 触发条件

1. 新复盘文档完成后进入治理评审。
2. 里程碑治理巡检。

频率守卫：
1. 默认按里程碑做一次 sweep，不按每篇复盘单独升级。

## 输入

1. `docs/retros/` 下一个或多个复盘文档
2. 当前 `AGENTS.md`
3. 升级判定标准

## 升级判定标准

候选规则必须同时满足：
1. 可跨任务复用（非一次性）
2. 可操作且可观察
3. 能降低缺陷、返工或上下文污染
4. 能用简短稳定文本表达

## 执行流程

1. 提取候选实践。
2. 对每条候选标注：`promote` / `defer` / `reject`。
3. 对 `promote` 产出最小规则文本与落位建议。
4. 对 `defer/reject` 记录明确原因。
5. 校验 `AGENTS.md` 不混入项目进展叙述。

## 输出

1. 候选清单（含决策与原因）
2. `AGENTS.md` 建议补丁
3. defer backlog

## 完成标准

1. 升级规则简洁且可验证。
2. defer/reject 有清晰理由。
3. `AGENTS.md` 仍保持“规则文档”定位。
